# Track D — Destructive Operations & Filesystem Safety

## Summary
The marketplace ships several shell and Node scripts that mutate state **outside the repo**: `~/.claude/skills/`, the user's project CWD (wrangler/nuxt/vite configs), `~/.local/bin/`, and `~/wp-content/plugins/`. The most serious issue is that `scripts/install-skill.sh` derives the `rm -rf` target from an unvalidated `$1` argument and only guards against the empty case — `name="."`, `".."`, and `"foo/../.."` survive the `[ -z ]` check and can drive a recursive delete at or above `$HOME/.claude/skills/`. A second systemic issue is that JSONC-aware scripts (cloudflare-durable-objects `migration-generator.sh`, cloudflare-workers-observability `setup-logging.sh`) edit the user's `wrangler.jsonc` in place using tools (`grep -v '^\s*//'` + `jq`, or `jq` directly) that **silently strip all comments and reject inline-commented JSONC**, leaving the file corrupted or comment-free. Most other in-place edits do create backups, but a handful (setup-logging.sh) `mv` a temp file over the original without any backup and without `set -e` safety on the failure path.
