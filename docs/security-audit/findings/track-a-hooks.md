# Track A — Hook & Auto-Execution Scripts

## Summary

Audited 7 hook scripts and 2 hook manifests across 3 plugins (`bun`, `nuxt-seo`, `claude-code-bash-patterns`). The highest-severity surface — the `dangerous-command-guard.py` PreToolUse guard — has a narrow `except (json.JSONDecodeError, KeyError)` clause that explicitly fails open (`sys.exit(0)`, allow) on malformed-but-valid JSON, and additionally crashes with an uncaught `AttributeError` (exit 1) on `[]`/`null`/string/scalar input because the except clause doesn't cover `AttributeError`; the bypass list is large (`rm -fr /`, `rm --recursive --force /`, `find / -delete`, `git push origin main --force`, `curl | sh`, `chmod -R 777 /usr`, etc.). The `bash-audit-logger.sh` and `package-manager-enforcer.sh` both read input from `$CLAUDE_TOOL_INPUT` instead of stdin — meaning under the documented Claude Code PreToolUse contract (JSON on stdin) neither script receives any input and they silently no-op (the enforcer can never block, the logger records nothing for the actual command). The SessionStart `detect-nuxt-seo.sh` (matcher `*`, runs every session) is well-quarantined by hardcoding module names, but uses unquoted `${MODULES[*]}` and `'$SITE_URL_SET'` in the env write and omits `>> "$CLAUDE_ENV_FILE"` atomicity; the realistic injection path is content of `nuxt.config.ts` manipulating the `awk` `SITE_URL_SET` variable, but that value is constrained to `"true"`/`false`. **Counts: 2 Critical, 3 High, 4 Medium, 3 Low, 2 Info = 14 findings.**
