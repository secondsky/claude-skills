# Track B — Subprocess Spawning & Config-Driven Execution

## Summary

`claude-skills` has only a small number of real subprocess call sites in executable code (most `exec`/`curl`/`eval` grep hits are inside documentation strings, Cloudflare Durable Object `.sql.exec()` calls, Playwright DOM `$eval`, or HTTP-only `curl` requests). The most severe finding is in `plugins/mcp-dynamic-orchestrator/src/orchestrator.ts:147`, where `spawn(entry.mcp.command, entry.mcp.args ?? [], …)` runs an arbitrary command read from a JSON registry file with **no command allowlist, no argument validation, and no transport restriction**; any plugin manifest (or any edited `mcp.registry.json`) that lands on disk drives unconditional command execution the first time `describe_mcp` or `execute_mcp_code` touches that entry. The rest are fixed-string `execSync('bun install')`/`execSync('npm install')` calls (benign), two `eval "$cmd"` shell scripts that interpolate unsanitized CLI flags into a `wrangler tail` command (medium risk), and one cosmetic `curl | bash` that appears only inside an error message string (not executed). No checksum-pinned remote-execution flow was found.

## Inventory of subprocess calls

- `plugins/mcp-dynamic-orchestrator/src/orchestrator.ts:3` — `import { spawn } from "node:child_process"` — **risky**: consumed by the `spawn` call below.
- `plugins/mcp-dynamic-orchestrator/src/orchestrator.ts:147` — `spawn(entry.mcp.command, entry.mcp.args ?? [], { stdio, env: { ...process.env, ...entry.mcp.env } })` — **risky**: `command`, `args`, and `env` are all read verbatim from `mcp.registry.json` (`loadRegistry()` at line 74, `getEntry()` at 87) with no allowlist; array form is used (good), but the command token itself is unconstrained. Process inherits full `process.env` plus attacker-supplied `entry.mcp.env`.
- `plugins/playwright/skills/playwright/run.js:15` — `const { execSync } = require('child_process')` — benign (import).
- `plugins/playwright/skills/playwright/run.js:42` — `execSync('bun install', { stdio: 'inherit', cwd: __dirname })` — benign: fixed string, cwd pinned, shell-form.
- `plugins/playwright/skills/playwright/run.js:43` — `execSync('bunx playwright install chromium', { stdio: 'inherit', cwd: __dirname })` — benign: fixed string.
- `plugins/playwright/skills/playwright/run.js:45` — `execSync('npm install', { stdio: 'inherit', cwd: __dirname })` — benign: fixed string.
- `plugins/playwright/skills/playwright/run.js:46` — `execSync('npx playwright install chromium', { stdio: 'inherit', cwd: __dirname })` — benign: fixed string (note: `npx ...@latest` style fetches newest package from npm registry; first-run supply-chain risk, see B-006).
- `plugins/cloudflare-sandbox/skills/cloudflare-sandbox/scripts/test-sandbox.ts:45-46` — `const { execSync } = await import('child_process'); execSync('docker ps', { stdio: 'ignore' })` — benign: fixed string, used only as a health probe.
- `plugins/cloudflare-workers/skills/cloudflare-workers-dev-experience/scripts/dev-setup.sh:38` — `error "Bun is required. Install: curl -fsSL https://bun.sh/install | bash"` — benign: text inside an `error` message string; the `| bash` is **never executed** by the script. The script itself runs `bun install` (line 431) and `bunx wrangler types` (line 440) — both fixed.
- `plugins/cloudflare-workers/skills/cloudflare-workers-observability/scripts/analyze-logs.sh:150,175,189` — `eval "$cmd"` and `timeout "$duration" bash -c "$cmd"` where `$cmd` is built from `WORKER_NAME`, `ENV_NAME`, `STATUS_FILTER`, `METHOD_FILTER`, `SEARCH_TERM` — **medium risk**: shell-form `eval` with unquoted interpolation; values come from CLI args (`--worker`, `--status`, `--search`, etc.) with no sanitization.
- `plugins/ultracite/skills/ultracite/scripts/install-ultracite.sh:216` — `eval $INIT_CMD` — low risk: `INIT_CMD` is built entirely from fixed-string `case` branches over `PM` (detected, not user input).
- `plugins/ultracite/skills/ultracite/scripts/migrate-to-ultracite.sh:180,228,238` — `eval $INIT_CMD`, `eval "$REMOVE_CMD $ESLINT_PACKAGES"`, `eval "$REMOVE_CMD $PRETTIER_PACKAGES"` — low risk: same fixed-string `case` branches; appended package names are also fixed strings selected by `grep -q '<literal>' package.json`. Values are bounded but the `eval` pattern is still poor hygiene.
- All other `curl`/`wget` hits: HTTP-only requests (e.g., `curl -s … "$url"` in `check-versions.sh`, `test-upload.sh`, `verify-deployment.sh`, `security-audit.sh`) — out of scope for subprocess exec; reviewed, no shell-injection-into-`sh` flow.
- All `sql.exec(...)` hits in `cloudflare-durable-objects/`, `cloudflare-workers-testing/`, etc. are D1/Durable Object SQL bindings, not shell exec — out of scope.
- All `$eval`/`$$eval` in playwright helpers/templates are Playwright DOM evaluation helpers, not shell exec — out of scope.

Notes on what was checked and cleared:
- All `.sql.exec(` hits in `cloudflare-durable-objects/`, `cloudflare-workers-testing/`, `drizzle-orm-d1/` are D1/Durable-Object SQL bindings, not shell exec.
- All `$eval`/`$$eval` in Playwright templates/helpers are DOM-eval helpers scoped to the browser page, not host subprocess calls.
- `curl` invocations across the Cloudflare plugins (`check-versions.sh`, `test-upload.sh`, `verify-deployment.sh`, `security-audit.sh`, `create-service-token.sh`, `test-access-jwt.sh`) are HTTP-only requests; none pipe into a shell.
- `yq eval '.'` in `github-project-automation/scripts/validate-workflows.sh` is YAML parsing, not shell eval.
- No `subprocess`/`os.system` usage in any `.py` file under `plugins/` was found.
- No `child_process.execFile` usage was found; the codebase uses either `spawn` (orchestrator) or `execSync` (everywhere else).
