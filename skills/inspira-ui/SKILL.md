# Inspira UI - Animated Vue/Nuxt Component Library

Inspira UI is a collection of 120+ reusable, animated components powered by TailwindCSS v4, motion-v, GSAP, and Three.js ‚Äî crafted to help ship beautiful Vue and Nuxt applications faster.

## When to Use This Skill

Use Inspira UI when building:
- **Animated landing pages** with hero sections, testimonials, and effects
- **Modern web applications** requiring 3D visualizations and interactive elements
- **Marketing sites** with eye-catching backgrounds and text animations
- **Dashboards** with animated data visualizations and progress indicators
- **Portfolio sites** with image galleries, carousels, and showcase effects
- **Interactive experiences** with custom cursors, special effects, and particle systems
- **Forms and inputs** with modern animated effects and validation
- **Vue 3 or Nuxt 4 projects** requiring production-ready animated components

**Key Benefits**:
- 120+ copy-paste components (not a traditional library)
- Full TypeScript support with Vue 3 Composition API
- TailwindCSS v4 with OkLch color space
- Responsive and mobile-optimized
- Free and open source (MIT license)
- Production-tested components
- No external dependencies required (copy-paste model)

## Installation & Setup

### Prerequisites

Ensure you have:
- **Vue 3** or **Nuxt 4** project
- **TailwindCSS v4** (required - for TailwindCSS v3, see [Inspira UI v1](https://v1.inspira-ui.com))
- **Node.js 18+** recommended
- **Bun** (recommended) or npm/pnpm

### Step 1: Install TailwindCSS v4

Follow the official [TailwindCSS installation guide](https://tailwindcss.com/docs/installation).

### Step 2: Install Core Dependencies

Install utility libraries and TailwindCSS animation utilities:

```bash
bun add -d clsx tailwind-merge class-variance-authority tw-animate-css
```

Install VueUse and Motion-V for animations:

```bash
bun add @vueuse/core motion-v
```

Follow the [Motion-V Vue/Nuxt setup guide](https://motion.dev/docs/vue) to configure motion-v in your project.

## Using Components

1. **Browse components** at [https://inspira-ui.com/components](https://inspira-ui.com/components)

  ```bash
  bun add gsap
  ```

Check each component's documentation for specific requirements.

## Code Style Guidelines

When using or modifying Inspira UI components:

### Props Definition

Use TypeScript interfaces with `defineProps`, **not** object syntax:

```vue
<script setup lang="ts">
// ‚úÖ DO THIS
interface Props {
  title: string;
  count?: number;
  isActive?: boolean;
}

const props = withDefaults(defineProps<Props>(), {
  count: 0,
  isActive: false,
});

// ‚ùå DON'T DO THIS
const props = defineProps({
  title: { type: String, required: true },
  count: { type: Number, default: 0 },
});
</script>
```

### Explicit Imports

Always include explicit imports (even if using Nuxt auto-imports):

```vue
<script setup lang="ts">
// Always import explicitly for Vue.js compatibility
import { ref, onMounted, computed } from "vue";
import { useWindowSize } from "@vueuse/core";
</script>
```

### Constants Naming

Use `CAPS_SNAKE_CASE` for constants with component prefix:

```typescript
const AURORA_GRADIENT_COLORS = ["#ff0000", "#00ff00", "#0000ff"];
const BUTTON_ANIMATION_DURATION = 300;
```

### Avoid Enums

Use `as const` objects instead of Enums:

```typescript
// ‚úÖ DO THIS
export const ButtonVariants = {
  Primary: "primary",
  Secondary: "secondary",
  Destructive: "destructive",
} as const;

export type ButtonVariant = typeof ButtonVariants[keyof typeof ButtonVariants];

// ‚ùå DON'T DO THIS
enum ButtonVariants {
  Primary = "primary",
  Secondary = "secondary",
}
```

### Component File Structure

```vue
<template>
  <!-- Template code -->
</template>

<script setup lang="ts">
// Imports
import { ref, computed } from "vue";

// Props interface
interface Props {
  // ...
}

// Props definition
const props = withDefaults(defineProps<Props>(), {
  // defaults
});

// Emits
const emit = defineEmits<{
  click: [value: string];
}>();

// Component logic
const state = ref(0);
</script>

<style scoped>
/* Scoped styles if needed */
</style>
```

## Common Issues & Solutions

### Issue 1: TailwindCSS v4 Not Working

**Problem**: Components not styling correctly, CSS variables undefined

**Solution**:
- Verify TailwindCSS v4 is installed: `bun add tailwindcss@next`
- Check `@import "tailwindcss"` is in your CSS
- Ensure CSS variables are defined in `:root` and `.dark`
- Verify `@theme inline` block is present

### Issue 2: Motion-V Animations Not Working

**Problem**: Motion components render but don't animate

**Solution**:
- Install motion-v: `bun add motion-v`
- Follow [Motion-V setup guide](https://motion.dev/docs/vue)
- For Nuxt, ensure proper plugin configuration
- Import Motion component: `import { Motion } from "motion-v"`

### Issue 3: Three.js Components Not Rendering

**Problem**: 3D components (Globe, Cosmic Portal, etc.) show blank

**Solution**:
- Install Three.js: `bun add three @types/three`
- Wrap in `<ClientOnly>` for Nuxt
- Check browser console for WebGL errors
- Verify canvas element is rendering

### Issue 4: OGL Components Not Working

**Problem**: OGL-based components (Fluid Cursor, Liquid Background) not displaying

**Solution**:
- Install OGL: `bun add ogl`
- Ensure WebGL support in browser
- Wrap in `<ClientOnly>` for SSR
- Check for shader compilation errors in console

### Issue 5: Props Type Errors

**Problem**: TypeScript errors with component props

**Solution**:
- Use interface-based props, not object syntax
- Use `withDefaults()` for default values
- Ensure explicit imports for types
- Check props are correctly typed in parent component

### Issue 6: Icons Not Showing

**Problem**: Icon components render empty

**Solution**:
- Install Iconify Vue: Follow [Iconify guide](https://iconify.design/docs/icon-components/vue/)
- Or replace `<Icon>` with your preferred icon library
- Or use SVG icons directly

### Issue 7: CSS Variables Not Applied

**Problem**: Colors or sizing not working as expected

**Solution**:
- Verify `:root` CSS variables are defined
- Check `.dark` variants are set up
- Ensure `@theme inline` block includes all variables
- Use `var(--variable-name)` syntax or Tailwind classes

### Issue 8: Component Dependencies Missing

**Problem**: Component imports failing or features not working

**Solution**:
- Check component documentation for specific dependencies
- Install required packages (Three.js, OGL, GSAP, etc.)
- Verify peer dependencies are satisfied
- Check for component-specific setup requirements
